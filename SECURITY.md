# Security Policy

## Supported Versions

Only the latest minor release line is eligible to receive security updates. Security fixes are not guaranteed for previous minor or major versions.

| Version | Supported |
| -------- | --------- |
| Latest minor release | :white_check_mark: |
| Previous minor releases | :x: |
| Previous major releases | :x: |

For example, if the current release is `1.4.x`, only `1.4.x` is supported. When `1.5.x` is released, support for `1.4.x` ends.

Users are expected to upgrade to the latest available minor version to receive security fixes.

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in ssherpa, please report it privately and do not disclose it publicly until the issue has been investigated and addressed.

Security reports should include:

- A description of the vulnerability
- Steps to reproduce the issue
- The affected version(s)
- Any proof-of-concept code or screenshots, if applicable

The project maintainers will make a reasonable effort to acknowledge receipt of valid vulnerability reports within 7 business days.

After review, vulnerabilities may be accepted, declined, or classified as informational. If a vulnerability is confirmed, a fix may be developed and released in a future version at the maintainers' discretion.

Please do not open public GitHub issues for undisclosed security vulnerabilities.
